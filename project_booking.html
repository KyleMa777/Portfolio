UofT Resource Booking: Redesigning for Efficiency

Transforming a cluttered, manual-search experience into a requirements-based interface that eliminates repetitive actions and significantly reduces cognitive load.

Role UX Research, Interaction Design, Prototyping
Context Course Project
Timeline 4 weeks · 2024

Overview

The UofT Resource Booking System is a critical tool for thousands of students, yet the legacy interface presents severe usability friction. This project applied Don Norman’s design principles and Nielsen’s usability heuristics to rebuild the system from the ground up.

By identifying the root causes of user frustration—specifically the high cognitive load required for manual searching. I transitioned the portal from a static grid to an intelligent, recommendation-driven platform.

The Problem: High Cognitive Load & Friction

Heuristic evaluations and user testing identified three critical pain points that made the original system "frustrating and inefficient":

01

Static Table Search

Availability was presented in a static table, forcing users to scan manually for open slots—a tedious process during peak hours.

02

Navigation & Data Reset

Switching between campus buildings reset the date to the current day, forcing users to repeatedly re-enter search criteria.

03

Low Discoverability

Key interface elements, such as the booking buttons and area selection tools, were "hard to notice" due to a cluttered layout.

The Solution: Needs-First Design

Guided by A/B testing insights, the redesign focuses on three core pillars to simplify the user journey:

01

Automated Querying

Allows users to input requirements first, eliminating manual table scanning.

02

Persistent Filters

Filter-based system that maintains progress, reducing repetitive actions.

03

Waitlist & Notifications

Added a "Notify Me" feature for unavailable slots, with real-time alerts.

Optimized Booking Workflow

The redesigned end-to-end workflow streamlines the user journey from initial search to final confirmation. User testing confirmed that this optimized flow saved users an average of 53.8 seconds compared to the original system.

Enhanced Visibility & Signifiers

Drawing from Don Norman’s principles, the new design improves the visibility of system status and provides clear signifiers for user actions. The system ensures users always understand the current state of their requests.

The results from our A/B tests were definitive: 100% of testers preferred the new design.

Reflection

This project demonstrated how Design Thinking can simplify complex systems. By focusing on discoverability and mapping, the redesign successfully improved the overall study room booking experience at UTM.
